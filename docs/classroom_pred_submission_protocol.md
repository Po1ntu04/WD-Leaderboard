# 课堂 `pred.txt` 提交协议

本文档用于发给同学，说明课堂统一分词提交流程、文件命名、格式标准和常见错误。

---

## 一、提交流程

1. 课上统一发放 `raw.txt`
2. 同学使用自己的算法对 `raw.txt` 逐行分词
3. 生成预测结果文件 `pred.txt`
4. 按命名规则重命名后提交给教师
5. 本地统一评分与展示排行榜

---

## 二、文件命名规则

提交文件名必须为：

```text
{学号}_{姓名}_pred.txt
```

示例：

```text
2023123456_张三_pred.txt
2023123499_李四_pred.txt
```

要求：

- 学号必须写在最前面
- 学号与姓名之间用下划线 `_`
- 文件后缀必须是 `.txt`
- 文件名中不要加入空格、括号、版本号、最终版、修订版等额外内容
- 教师侧批量评测工具会优先按该命名规则识别并显示提交名

不推荐示例：

```text
张三_pred.txt
2023123456张三.txt
2023123456_张三_最终版_pred.txt
第八组_pred.txt
```

## 三、分词内容格式标准

### 1. 每一行对应 `raw.txt` 中的一行

- `raw.txt` 有多少行，提交文件就必须有多少行有效预测
- 不可漏行
- 不可多行
- 不可打乱顺序

### 2. 规范输出格式

每行必须写成：

```text
词1 / 词2 / 词3
```

说明：

- 分词之间使用空格包围的斜杠：` / `
- 不要混用中文顿号、逗号、竖线等其他分隔符
- 虽然系统兼容部分旧格式，但提交时必须按标准格式书写

### 3. 兼容性说明（用于避免误判）

教师侧评分程序当前兼容以下常见写法：

- 空格分词：

```text
词1 词2 词3
```

- 斜杠分词：

```text
词1 / 词2 / 词3
```

- 斜杠两侧存在不同数量空格的情况
- 纯空白分隔中出现多个连续空格的情况

但请注意：

- **兼容不等于推荐**
- 为减少格式错误、解析歧义和人工检查成本，**提交时仍统一要求使用 `词 / 词 / 词` 标准格式**
- 不建议使用 tab、全角空格、混合分隔符、额外说明文字等“虽然可能被兼容、但容易出错”的写法

### 4. 标点必须单独切分

例如：

```text
今天 / 发布 / 了 / 。
```

而不是：

```text
今天 / 发布 / 了。
```

### 5. 分词结果必须能还原原句

也就是说，把一行中的所有词直接拼接起来，必须与对应 raw 原句完全一致。

例如 raw 为：

```text
我爱自然语言处理。
```

则合法示例：

```text
我 / 爱 / 自然语言 / 处理 / 。
```

非法示例：

```text
我 / 爱 / 自然 语言 / 处理 / 。
```

如果存在多余空格、缺字、错字、漏标点、额外字符，都会导致评测失败或格式错误。

## 四、可选 runtime 展示行

如果希望展示算法运行时间，可以在文件最后一行加入一行元信息：

```text
# runtime_seconds: 0.183
```

注意：

- 这一行是**可选项**
- 使用 `#` 开头
- 只能放在文件最后一行
- 如果格式写错，教师侧会按格式错误处理

推荐完整示例：

```text
今天 / 发布 / 了 / 。
我们 / 开始 / 上课 / 。
# runtime_seconds: 0.183
```

当前教师侧程序对最后一行 runtime 元信息做了兼容处理，以下常见写法都能识别：

```text
# runtime_seconds: 0.183
# Runtime_seconds: 0.183
# runtime_seconds 0.183
# runtime_seconds: 0.183 // include loading time
```

但为了减少歧义，仍建议统一使用：

```text
# runtime_seconds: 0.183
```

## 五、常见错误提醒（请务必避免）

以下情况极易导致评分失败、格式错误或无法展示：

### 1. 行数不匹配

- 少一行
- 多一行
- 额外插入空白说明文字

### 2. 不按顺序输出

- 不允许重排句子顺序

### 3. 分词结果无法还原原句

常见原因：

- 漏字
- 多字
- 改写原文
- 标点丢失
- 插入额外空格或说明文字

### 4. 标点未单独切分

例如把 `了。` 写成一个词，会影响评分。

### 5. 文件命名不合规

若文件名不符合规则，教师侧整理和导入会受到影响。

### 6. 文件编码或内容异常

请确保：

- 使用 UTF-8 文本编码
- 不要在文件中加入奇怪控制字符
- 不要插入图片、表格、乱码、富文本内容

---

## 六、教师侧安全与稳健处理说明

为避免异常文件影响评测与展示，教师侧系统会采取以下保护措施：

- 只接收 `.txt` 文本文件
- 对文件大小做限制
- 对非法控制字符做拦截
- 仅识别最后一行的 `# runtime_seconds: ...` 元信息
- 校验每行是否能还原原句
- 校验总有效行数是否与 `raw.txt` 一致
- 排行展示时对文本做转义，避免页面显示异常

若提交文件故意加入异常内容、干扰内容或破坏展示流程，教师可直接判为无效提交。

---

## 七、建议的提交前自查清单

提交前请确认：

- [ ] 文件名为 `{学号}_{姓名}_pred.txt`
- [ ] 文件是 `.txt`
- [ ] 每行对应 raw 的同一行
- [ ] 总有效行数一致
- [ ] 规范格式为 `词 / 词 / 词`
- [ ] 标点单独切分
- [ ] 拼接后能还原原句
- [ ] 没有额外说明文字混入预测内容
- [ ] 若写 runtime，则放在最后一行，格式为 `# runtime_seconds: 数值`
